76.谈谈你对SpringBoot的理解，它有哪些特性

​ 快速开发Spring应用的脚手架，简化Spring的初始搭建及配置
​ 1.无配置集成，开箱即用
​ 2.基于JavaConfig
​ 3.内置Web容器，无需web.xml，jar文件就能启动web应用
​ 4.管理了第三方依赖版本，减少版本冲突
​ 5.自带监控功能
​ 参考

==79.Springboot的自动配置原理==

​ 1.@SpringBootApplication
​ 导入@SpringBootConfiguration标注当前类是配置类
​ 导入@EnableAutoConfiguration，自动配置
​ 2.@EnableAutoConfiguation引入了@Import
​ 3.Spring容器启动(SpringBoot.run)加载IOC容器会解析@Import注解
​ 4.@Import导入了一个DeferredImportSelector
​ 会使Springboot的自动配置类顺序在最后，方便我们扩展和覆盖
​ @ConditionalOnBean
​ 配置了某个Bean，就帮我们把剩下的组件配置好
​ @ConditionalOnMissingBean
​ 如果我们没有配置，就自动帮我们配置，如DispatcherServlet
​ 5.读取所有/META-INF/spring.factories文件(类似API)
​ 6.过滤出所有AutoConfigurationClass(初筛)
​ 7.通过@Conditional排除无效的自动配置类

==80.为什么SpringBoot的jar可以直接运行==

​ 1.spring-boot-maven-plugin可以把程序打包成一个可执行的jar包
​ 2.打包后，生成了一个Fat jar，包含了应用以来的jar包和Springboot loader相关的类
​ 3.java -jar会去jar中(jvm规范)找MANIFEST.MF，找真正的启动类(Main-Class)
​ 4.Fat jar的启动函数是JarLauncher，它负责创建一个LaunchedURLClassLoader来加载lib下面的jar，并以一个新线程去启动应用启动类的Main函数(MANIFEST.MF中的Start-Class)

==81.SpringBoot的启动原理==

​ 1.运行main方法，构造函数初始化信息，从spring.factories读取监听器，初始化器
​ 2.运行run方法，读取环境变量、配置信息
​ 3.创建springApplication上下文：ServletWebServerApplicationContext
​ 4,预初始化上下文
​ 将启动类作为配置类进行读取，将配置注册为BeanDefinition
​ 5.调用refresh加载ioc容器
​ invokeBeanFactoryBeanProcessor
​ @Import加载所有自动配置类
​ onRefresh：常见内置servlet容器
​ 6.在这个过程中springboot会调用很多监听器对外进行扩展

==82.SpringBoot内置Tomcat原理==

​ 1.依赖spring-boot-starter-web依赖会在Springboot中添加ServletWebServerFactoryAutoConfiguration servlet自动配置类
​ 2.该自动配置类通过@Import导入了一个Web容器工厂(默认tomcat)
​ 3.在加载IOC容器时，解析@Import，onRefresh(会拿到tomcat容器工厂)创建内置tomcat并启动
​ new tomcat()
​ 启动tomcat
​ 挂起线程
​ SpringMVC中的DispatcherServlet也是自动配置类导入进来的

==83.SpringBoot外部tomcat启动原理==

​ SPI
​ 当servlet容器启动时候，就会去META-INF/services 文件夹中找到javax.servlet.ServletContainerInitializer
​ 执行官方的SpringServletContainerInitializer(继承了ServletContainerInitializer)
​ 执行自己的SpringBootServletInitializer#onStartup(继承了WebApplicationInitializer)
​ 启动SpringBoot

==84.自定义Starter的过程==

​ 导入spring-boot-configuration-processor，配置文件绑定就会有提示
​ 可以自定义XXXProperties文件，外部使用就能直接配置在.properties，.yml文件中
​ 自定义XXXAutoCOnfiguration自动导入(开启)配置类
​ 一定要在resources文件夹下创建META/spring.factories指定自动配置类
​ maven install

88.微服务架构的优缺点

​ 优点
​ 分工协作
​ 提高开发效率和敏捷性
​ 单个服务启动快
​ 专人处理专事，专注自己的服务
​ 并发能力
​ 充分利用服务器资源(并发量高的模块多部署几台服务器)
​ 维护能力
​ 根据功能垂直拆分，责任更加分明，维护更加精准
​ 容错
​ 弱依赖的服务出现故障，可以进行熔断(隔离)依然不影响主业务正常使用
​ 扩展
​ 新服务可以采用任意新技术
​ 缺点
​ ==分布式==
​ 分布式系统较难编程(远程调用速度很慢，并且总是面临失败的风险)，对开发人员技术要求高
​ ==最终一致性==
​ 对于分布式系统，保持强一致性非常困难，所以每个人都必须管理最终一致性
​ ==运维复杂度==
​ 隐式接口
​ 服务和服务通过接口来联系，当一个服务更改接口格式时，所有涉及此接口的服务都要做调整
​ 重复劳动
​ 很多服务用到同一功能，但这一功能并没有大到提供一个服务的程度，不同服务团队都会单独开发这一功能
